Foundation

Three irreducible primitives

Every autonomous system that makes decisions needs exactly three things. Remove any one and the system is ungovernable.

Historical precedent: foundational systems often reduce to a small set of irreducible primitives (e.g., relational algebra, public-key exchange, hash chaining). We propose an analogous reduction for autonomous system governance.

Governance — Constraint Enforcement

Is this action allowed?

The governance primitive evaluates behavioural constraints deterministically before an action executes. No neural network in the enforcement path. The agent proposes, governance disposes.

Properties:

  • Pre-execution: constraints are checked before the action, not after
  • Deterministic: same input, same decision, every time
  • External to agent: the agent cannot modify its own governance boundary
  • Domain-agnostic: the constraint language changes, the enforcement mechanism does not

Protocols: Ethics Gate (EGP-1.0), Consent Ledger (CNL-1.0)

Reasoning — Soundness Verification

Is this reasoning sound?

The reasoning primitive makes the inferential structure of every decision inspectable. Each step in the chain — observation, derivation, assumption, decision — is individually verifiable.

Properties:

  • Structural: exposes the reasoning graph, not just the conclusion
  • Assumption-explicit: every assumption is named and trackable
  • Bias-detectable: cognitive patterns (anchoring, confirmation, availability) are flagged
  • Composable: reasoning chains can be compared across agents or decisions

Protocols: Cognitive Ledger (CLP-1.0), Assumption Registry (ARP-1.0), Dispute Protocol (DSP-1.0)

Traceability — Decision Recording

What happened and can we prove it?

The traceability primitive creates a tamper-evident, append-only record of every decision. SHA-256 hash-chaining. Each record includes the hash of its predecessor. Modify any record and every downstream hash breaks.

Properties:

  • Append-only: records can be added, never modified or deleted
  • Hash-chained: SHA-256, each record references its predecessor's hash
  • Tamper-evident: any modification is detectable by recomputing the chain
  • Forensic: supports post-hoc reconstruction of complete decision history
  • Not a blockchain: linear chain, no distributed consensus, no network required

Protocols: Clearpath (CAP-1.0), Harm Trace (HTP-1.0), Trust Score (TSP-1.0)

We propose that these three primitives are independently necessary and jointly sufficient for governing autonomous systems. The claim of irreducibility is empirical — validated across 8 protocols, 297 tests, and 8 domains — and is proposed for formal mathematical proof.

For researchers

Formal properties

Threat Model

OMEGA assumes the governance runtime is external to and trusted by the agent. The agent cannot modify its own governance constraints. The trusted computing base (TCB) of OMEGA is the governance runtime and the integrity of SHA-256. If an agent controls its own governance runtime, enforcement can be circumvented — this is an explicit design boundary, not a limitation. OMEGA assumes a single-writer append-only model. Byzantine distributed consensus is out of scope.

What OMEGA prevents:

  • Unauthorised actions executing (governance primitive)
  • Reasoning chains being opaque or uninspectable (reasoning primitive)
  • Decision records being modified, deleted, or reordered (traceability primitive)

What OMEGA does not claim to prevent:

  • Covert communication between agents (steganographic collusion)
  • Adversarial agents that control their own runtime
  • Side-channel attacks on the governance evaluation
  • Attacks on the hash function itself (relies on SHA-256 collision resistance)

Current scope: single-agent governance. Multi-agent composition — specifically, how governance constraints compose when agents with different trust boundaries interact — is an open research question.

Hash Chain Specification

Algorithm: SHA-256

Structure: Linear chain (not a Merkle tree, not a DAG)

Each record: { data, timestamp, previousHash, hash }

hash = SHA-256( canonical_json(data) + timestamp + previousHash )

Encoding: canonical JSON serialisation (key-order invariant)

Integrity guarantee: If any record in the chain is modified, its hash changes, which invalidates the hash of every subsequent record. Verification requires recomputing the full chain from genesis.

What the hash chain provides:

  • Tamper evidence (detection of any modification)
  • Ordering (each record has exactly one predecessor)
  • Append-only semantics (new records extend, never replace)

What the hash chain does not provide:

  • Confidentiality (records are plaintext)
  • Non-repudiation (no digital signatures on records)
  • Distributed consensus (single-writer assumption)
  • Zero-knowledge properties (verifier sees all data)
  • Author authentication (the hash chain provides integrity and tamper evidence, not author authentication — no signatures in v1)

Potential extensions under consideration: ZK proofs for governance compliance verification without revealing constraints; digital signatures for multi-party audit scenarios; Merkle tree structure for efficient partial-chain verification.

Governance Enforcement Model

The governance primitive operates as a synchronous gate. The execution flow is:

  1. Agent proposes action
  2. Governance runtime evaluates action against constraint set
  3. If all constraints are satisfied → action executes
  4. If any constraint is violated → action blocked, violation recorded
  5. Decision (allow/block) is appended to the hash chain

Properties:

  • Synchronous: no action executes without governance evaluation
  • Fail-closed: if governance cannot evaluate, default is block
  • Deterministic: constraint evaluation has no randomness, no learned components
  • Auditable: every governance decision is recorded with full context

The constraint language is domain-specific. The enforcement mechanism is domain-agnostic. A defence ROE constraint and a financial compliance constraint use different policy languages but identical enforcement architecture.

Relationship to Multi-Agent Security

Recent work in multi-agent security identifies threats that emerge from interactions between AI agents — including steganographic collusion, illusory attacks, and coordinated swarm exploitation. A key finding: anomaly detection alone is fundamentally insufficient against adversaries that can optimise for undetectability.

OMEGA's governance primitive takes the complementary approach: instead of detecting bad behaviour after the fact, it prevents unauthorised actions from executing before the fact. This is architectural enforcement, not behavioural monitoring.

Current limitations:

  • OMEGA governs individual agents, not interactions between agents
  • Cross-agent governance composition is an open research problem
  • OMEGA does not address covert communication channels between agents

The intersection of deterministic governance enforcement and multi-agent security — specifically, how governance constraints compose across trust boundaries — represents a significant open research direction.

Relationship to Quantitative Risk Modelling

Recent work on AI risk management argues that verifiable, provably-safe AI architectures are urgently needed to supply deterministic evidence where current models are the result of opaque end-to-end optimisation.

OMEGA provides exactly this deterministic evidence layer:

  • Every governance decision is recorded with full context
  • Every reasoning chain is structurally inspectable
  • Every record is tamper-evident and independently verifiable

This enables quantitative risk assessment based on architectural guarantees rather than behavioural observation — deterministic evidence that a constraint was evaluated and satisfied, not probabilistic confidence that a model will behave.

Research frontier

Open questions

These are the problems we have not solved. They represent the frontier of this work.

  1. Irreducibility proof — Can it be formally proven that governance, reasoning, and traceability are the minimal complete set? What mathematical framework is appropriate?
  2. Multi-agent composition — How do governance constraints compose when multiple agents with different trust boundaries interact? What are the sufficient conditions for safe composition?
  3. Information-theoretic limits — What guarantees are provable for each primitive? Where are the fundamental limits of deterministic governance enforcement?
  4. Zero-knowledge governance — Can an agent prove it operated within its governance constraints without revealing what those constraints are? What proof systems are suitable?
  5. Adversarial robustness — Under what threat models does external governance enforcement remain sound? What side-channel attacks are possible?
  6. Formal verification — Can the governance enforcement mechanism itself be formally verified? What properties should the verification target?
  7. Trusted execution assumptions — What hardware or isolation guarantees are required to prevent governance bypass in cyber-physical deployments?

If you are working on any of these problems, we would like to hear from you.

Abstract: This paper identifies three irreducible computational primitives required for governing autonomous AI systems: governance (constraint enforcement), reasoning (soundness verification), and traceability (decision recording). We present implementations across eight protocols validated by 297 formal tests and deployed across eight domains including defence, finance, manufacturing, and clinical environments. We argue these primitives are independently necessary and jointly sufficient, and we position this work relative to historical computational primitives including Codd's relational model, Merkle's hash chain, and Diffie-Hellman's key exchange.
